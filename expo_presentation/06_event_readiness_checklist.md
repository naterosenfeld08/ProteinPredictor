# Event Readiness Checklist

## 1) Print Legibility Checks

- [ ] Board title readable from 6 feet.
- [ ] Body text readable from 3-4 feet.
- [ ] All figures include axis labels and units where applicable.
- [ ] Color choices remain distinct in grayscale print.
- [ ] Every numeric claim has a matching source file in repo.
- [ ] QR code scans correctly from phone camera at 2-3 feet.

## 2) Trifold Layout Checks

- [ ] Left panel = background/methods only.
- [ ] Center panel = largest visuals and key findings.
- [ ] Right panel = interpretation, limitations, next steps.
- [ ] Visual flow is left-to-right without crossing arrows.
- [ ] Footer includes repo link and contact.

## 3) Hardware and Demo Checks

- [ ] Laptop power supply packed.
- [ ] HDMI/USB-C adapter tested.
- [ ] External monitor resolution/scaling set to readable mode.
- [ ] Offline fallback files available locally (PNG + PDF).
- [ ] Slide loop tested for 5+ minutes without interaction.
- [ ] Brightness and glare acceptable under venue lighting.

## 4) Content Integrity Checks

- [ ] Discovery numbers match `figure_data.json`.
- [ ] Pilot metrics match `figure_data.json`.
- [ ] Method labels match implemented code behavior.
- [ ] Limitations section present and honest.
- [ ] Next steps are specific and feasible.

## 5) Day-Of Runbook (15 minutes before judging)

1. Mount board and center main results figure.
2. Connect monitor and start slide/figure loop.
3. Open fallback screenshots and summary file on laptop.
4. Rehearse 30-second hook once out loud.
5. Confirm your one-sentence novelty statement is memorized.
